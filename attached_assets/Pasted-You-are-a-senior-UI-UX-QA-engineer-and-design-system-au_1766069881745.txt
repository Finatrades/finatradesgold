You are a senior UI/UX QA engineer and design-system auditor.

Goal:
Perform a FULL Light Mode + Dark Mode verification across the entire platform.
Ensure CSS layout, spacing, typography, modals, forms, content colors, and UI states look perfect.
Think like a normal user + designer + accessibility reviewer.

Scope:
ALL pages, ALL components, ALL popups/modals, ALL forms, ALL tables, ALL cards, ALL charts,
Admin + User + Corporate dashboards, and every feature module.

========================================================
1) GLOBAL THEME TOGGLE TEST
========================================================
A) Confirm there is a Light/Dark toggle (or system setting).
B) Switching theme must:
- Update instantly (no refresh required).
- Persist after reload (localStorage/cookie/profile setting).
- Persist across logout/login (if profile-based).
- Work on all routes (no page stuck in old theme).

C) Test theme toggle on:
- Desktop
- Tablet
- Mobile

========================================================
2) COLOR & CONTRAST “PERFECTNESS” CHECK
========================================================
For BOTH Light and Dark modes validate:
- Text contrast readable everywhere (titles, body text, muted text).
- Links are distinguishable (not same as normal text).
- Disabled text does not look like active text.
- Error/red, success/green, warning/yellow are visible and consistent.

Accessibility checks:
- Ensure minimum contrast for primary text vs background.
- Focus outlines visible in both modes (keyboard navigation).

========================================================
3) LAYOUT / SPACING / RESPONSIVENESS CHECK
========================================================
For BOTH themes validate:
- No broken alignment, overflow, clipped corners.
- Sidebar, header, cards, tables align correctly.
- Padding/margins consistent.
- No unexpected horizontal scroll.
- Modal centered and responsive.
- Z-index layering correct (modal over blur backdrop, dropdown over modal, etc.).

Test breakpoints:
- 1440px, 1024px, 768px, 375px

========================================================
4) COMPONENT-BY-COMPONENT VISUAL QA
========================================================
Test every component in BOTH modes:

A) Navigation
- Sidebar background, active item highlight, icons, dividers
- Top bar, search input, user avatar, notifications

B) Buttons
- Primary / Secondary / Outline / Ghost / Destructive
- Hover, Active, Disabled, Loading states
- Ensure button text always readable

C) Forms (MOST IMPORTANT)
Check all form controls:
- Input text color
- Placeholder color (must be visible but not too bright)
- Label color
- Border color
- Focus ring color
- Error state (border + message)
- Success state (if used)
- Dropdown/select menu background + item hover
- Checkbox + radio + toggles
- Date picker background, selected day, hover
- File upload box + file name display

D) Tables
- Header background vs rows
- Row hover color
- Zebra striping (if used) consistent
- Borders/dividers visible
- Status badges readable
- Pagination + filters readable
- Empty state and “No data” message

E) Modals / Dialogs / Popups
- Backdrop opacity correct in both themes
- Modal background not too bright/dark
- Close icon visible
- Form inside modal readable
- Dropdown inside modal not hidden behind modal (z-index)

F) Cards / Tiles / Widgets
- Card background vs page background contrast
- Shadow/glow not ugly in dark mode
- Icon colors consistent
- Numbers + captions readable

G) Badges / Chips / Tags
- Status chips (Approved/Pending/Rejected) readable
- Background + text contrast correct

H) Toasts / Alerts / Banners
- Toast background correct per theme
- Text readable
- Close button visible

I) Charts / Graphs (if any)
- Axes labels readable
- Gridlines not too faint
- Tooltips readable
- Legend readable

========================================================
5) CONTENT COLOR RULES (NO WEIRD MIX)
========================================================
Detect and flag:
- Dark text on dark background
- White text on white background
- Placeholder too faint
- Borders invisible
- Icons invisible
- Any component “stuck” in light colors while in dark mode (or vice versa)

========================================================
6) CSS IMPLEMENTATION CHECK (ENGINEERING)
========================================================
Audit CSS approach:
- Confirm all colors come from theme tokens/variables (NOT hardcoded hex scattered).
- Identify any hardcoded colors breaking theme.
- Ensure consistent use of:
  - CSS variables (e.g., --bg, --text, --muted, --border)
  - or Tailwind theme classes
- Ensure images/logos have dark-mode versions if needed.

========================================================
7) REQUIRED TEST PAGES / FLOWS
========================================================
Do theme QA on these real workflows (User + Admin):

User:
- Signup/Login/Forgot Password
- Dashboard
- Wallet (FinaPay)
- Vault (FinaVault)
- Buy Gold/Sell Gold screens
- Transfer/Transactions
- Certificates view/download
- KYC upload forms
- BNSL plan screens
- FinaBridge/trade screens (if user)

Admin:

- KYC Reviews
- Transactions
- Employees (Add Employee modal)
- Settings
- Reports

========================================================
8) OUTPUT REQUIRED (MUST DELIVER)
========================================================
Produce a “Light/Dark Mode QA Report” with:

1) PASS/FAIL summary for:
- Theme toggle
- Layout
- Forms
- Modals
- Tables
- Accessibility contrast

2) A bug list with:
- Page name
- Component name
- Theme (Light/Dark)
- Exact issue (e.g., “placeholder invisible”)
- Steps to reproduce
- Suggested fix (token/class name)

3) A “Hardcoded Color Audit” list:
- Where hardcoded hex values exist
- Which should be replaced by tokens

4) Final recommendation:
- “Theme quality is production-ready” OR “Needs fixes before launch”

IMPORTANT:
You must test by actually toggling themes and navigating across pages.
Do not assume it works just because the toggle changes background.
Verify every form input, modal, table, and text is readable and consistent.
