You are a senior frontend engineer.
Build a **FinaBridge Admin Panel** UI for the Finatrades platform.

Concept:
FinaBridge is a gold-backed trade finance module used by importers and exporters.
This ADMIN PANEL is for:
- Operations
- Compliance / Risk
- Trade Finance Desk

Admin users must be able to:
- Monitor all trade cases
- Inspect KYC summaries for importer/exporter
- Review documents and approvals
- See locked settlement value (gold) per case
- Approve / Reject / Hold trade cases
- Trigger “Release Settlement” (conceptual)
- View full audit trail

No backend yet → use local state + mock data.

Tech:
- React (Vite + React or Next.js)
- TypeScript
- TailwindCSS
- Optional: framer-motion for subtle animations

STATE & TYPES
────────────────────────

Use TypeScript interfaces.

```ts
type TradeCaseStatus =
  | "Draft"
  | "Awaiting Funding"
  | "Funded – Docs Pending"
  | "Under Review"
  | "Approved – Ready to Release"
  | "Released"
  | "Closed"
  | "Rejected"
  | "On Hold";

type TradeRole = "Importer" | "Exporter";

type PartyKycStatus = "Not Started" | "In Progress" | "Approved" | "Rejected";

type RiskLevel = "Low" | "Medium" | "High" | "Critical";

type TradeParty = {
  id: string;
  name: string;
  role: TradeRole;            // importer or exporter in that case
  country: string;
  kycStatus: PartyKycStatus;
  riskLevel: RiskLevel;
  sanctionsFlag: boolean;
};

type TradeCase = {
  id: string;
  reference: string;          // e.g. TF-2025-0007
  importer: TradeParty;
  exporter: TradeParty;
  commodityDescription: string;
  valueUsd: number;
  valueGoldGrams: number;
  lockedGoldGrams: number;    // reserved settlement amount
  status: TradeCaseStatus;
  createdAt: string;
  updatedAt: string;
  incoterm: string;           // FOB, CIF, etc.
  shipmentMethod: string;     // Air, Sea, Land
  expectedDeliveryDate: string;
  jurisdictionRisk: RiskLevel;
  amlFlags: string[];         // e.g. ["High invoice value", "High-risk country"]
};

type DocumentStatus = "Missing" | "Uploaded" | "Under Review" | "Approved" | "Rejected";

type TradeDocument = {
  id: string;
  caseId: string;
  type: string;               // Invoice, BL, CO, Insurance, etc.
  fileName: string;
  status: DocumentStatus;
  uploadedBy: string;
  uploadedAt: string;
};

type ApprovalStepStatus = "Pending" | "In Review" | "Approved" | "Rejected";

type ApprovalStep = {
  id: string;
  caseId: string;
  name: string;               // "Importer Verification", "Compliance Review", etc.
  role: string;               // "Ops", "Compliance", "Risk"
  status: ApprovalStepStatus;
  approverName?: string;
  decisionAt?: string;
  notes?: string;
};

type AuditLogEntry = {
  id: string;
  caseId?: string;
  actorName: string;
  actorRole: string;          // "Admin", "Compliance", "System"
  actionType: string;         // "StatusChange", "DocumentReview", "ReleaseFunds", etc.
  timestamp: string;
  details?: string;
  oldValue?: string;
  newValue?: string;
};

type LockedFundsSummary = {
  totalLockedGoldGrams: number;
  totalReleasedGoldGrams: number;
  lockedByStatus: Record<TradeCaseStatus, number>;
};
