You are a senior QA engineer, product auditor, and documentation specialist.

Your task is to REVIEW, TEST, and DOCUMENT the COMPLETE ADMIN PANEL
for the Finatrades gold-backed platform.

THINK LIKE A LAYMAN ADMIN.
Assume the admin is NOT technical and needs everything to be clear,
visible, and safe to operate.

------------------------------------------------
OBJECTIVES
------------------------------------------------
1. Verify that every admin screen works logically.
2. Confirm no missing data, broken flow, or hidden dependency.
3. Ensure all gold, wallet, payment, and approval actions are traceable.
4. Prepare a SIMPLE, STEP-BY-STEP ADMIN MANUAL in plain language.

------------------------------------------------
SYSTEMS TO COVER (ADMIN SIDE)
------------------------------------------------
- Admin Login & Roles
- User Management (Individual / Corporate)
- KYC / KYB Review
- FinaPay Wallet (USD + Gold grams)
- FinaVault (Gold ownership & storage)
- Buy Gold / Sell Gold approvals
- Wingold & Metals transaction verification
- BNSL Plans (Lock / Unlock / Maturity)
- FinaBridge (Trade Finance cases)
- Certificates (Ownership & Storage)
- Payments (Bank / Card / Crypto)
- Manual Transactions
- API Transactions (future-ready)
- Audit Logs
- Compliance & Risk Flags
- System Settings

------------------------------------------------
STEP 1: ADMIN ACCESS & SECURITY CHECK
------------------------------------------------
- Can admin log in smoothly?
- Are roles clearly separated?
- Can unauthorized users access admin pages? (must be NO)
- Is session timeout working?
- Is activity logged automatically?

Document findings clearly.

------------------------------------------------
STEP 2: DASHBOARD (LAYMAN VIEW)
------------------------------------------------
Check if dashboard clearly shows:
- Total users
- Total gold (grams)
- Total USD value
- Pending approvals
- Alerts / flags
- Recent activities

If a layman admin cannot understand the dashboard in 10 seconds,
FLAG IT.

------------------------------------------------
STEP 3: USER & KYC MANAGEMENT
------------------------------------------------
For each user:
- Can admin see full profile?
- Can admin see KYC status?
- Can admin approve / reject / request resubmission?
- Are reasons mandatory for rejection?
- Is every action logged?

Write instructions as:
"Click here → Check this → Approve / Reject → What happens next"

------------------------------------------------
STEP 4: FINAPAY WALLET CHECK
------------------------------------------------
For each wallet:
- USD balance visible?
- Gold grams visible?
- Transaction history complete?
- Manual adjustment option visible?
- Reason mandatory for any manual change?

Check:
USD ⇄ Gold conversion accuracy
No negative balance allowed

------------------------------------------------
STEP 5: FINAVAULT (GOLD OWNERSHIP)
------------------------------------------------
Verify:
- Gold ownership records per user
- Locked vs free gold
- Source of gold (Buy / Transfer / BNSL / Trade)
- Vault location reference
- Certificate linkage

If gold exists in wallet, it MUST exist in vault.

------------------------------------------------
STEP 6: BUY / SELL GOLD (MANUAL METHOD)
------------------------------------------------
Admin must see:
- Uploaded Wingold & Metals receipt
- Transaction amount
- Gold grams calculated
- Status (Pending / Approved / Rejected)

Test:
- Approval flow
- Rejection flow
- Partial errors

Document step-by-step.

------------------------------------------------
STEP 7: BNSL PLANS
------------------------------------------------
Check:
- Plan creation
- Gold locking
- Maturity rules
- Early exit logic
- Certificate update

Ensure admin can clearly see:
- Locked gold
- Release date
- Beneficiary

------------------------------------------------
STEP 8: FINABRIDGE (TRADE FINANCE)
------------------------------------------------
Admin must see:
- Trade case list
- Importer / Exporter
- Locked gold amount
- Documents uploaded
- Approval chain
- Final release action

No gold should move without admin approval.

------------------------------------------------
STEP 9: CERTIFICATES
------------------------------------------------
Verify:
- Digital Ownership Certificate
- Storage Certificate (Wingold & Metals)

Check:
- Auto-generation
- Download
- Update on balance change
- Unique certificate ID

------------------------------------------------
STEP 10: AUDIT LOGS & COMPLIANCE
------------------------------------------------
Every admin action MUST log:
- Who
- What
- When
- Before value
- After value

If any action is missing logs, FLAG IT.

------------------------------------------------
STEP 11: ERROR HANDLING (LAYMAN TEST)
------------------------------------------------
Test:
- What happens if data is missing?
- What happens if admin clicks wrong action?
- Are error messages simple and clear?

No technical errors should be shown to admin.

------------------------------------------------
STEP 12: PREPARE ADMIN PANEL MANUAL
------------------------------------------------
Create a FINAL ADMIN MANUAL with:
1. Admin login steps
2. Dashboard explanation
3. Daily admin tasks
4. Approval flows
5. Emergency actions
6. Do’s and Don’ts
7. Common mistakes & fixes

Language must be:
- Simple
- Clear
- Non-technical
- Step-by-step

------------------------------------------------
FINAL OUTPUT REQUIRED
------------------------------------------------
1. List of issues found (if any)
2. Missing features (if any)
3. Admin Panel Manual (ready to share as PDF)
4. Recommendations for improvement

DO NOT write code unless required.
FOCUS ON LOGIC, FLOW, SAFETY, AND CLARITY.
